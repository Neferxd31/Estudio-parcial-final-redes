Tema 1 — Tipos de Redes LAN / WAN

Las redes se clasifican según su tamaño geográfico, tecnología de transmisión y propósito. Este es el primer tema que sienta las bases para todo el curso.

¿Qué es una red?

"Una red de computadoras se establece cuando dos o más computadoras autónomas se conectan entre sí de forma permanente para compartir recursos e intercambiar información."

La palabra clave es autónomas: cada equipo funciona por sí solo, no depende del otro para encender o procesar. Por eso un mouse conectado al PC NO es una "red".

Clasificación de las redes

Por su ámbito (tamaño geográfico)

DistanciaUbicaciónTipo de red

Por su tecnología de transmisión

  • Redes broadcast (multipunto): todos comparten el mismo medio. Cuando uno transmite, todos escuchan. Ej: Ethernet.
  • Redes punto a punto: dos dispositivos comunicándose directamente sin compartir el medio con otros. Ej: enlace serial entre dos routers.

Redes LAN (Local Area Network)

Diseñadas desde el principio para transportar datos dentro de un área pequeña. El cableado es propiedad del usuario u organización.

Características

  • Generalmente broadcast (medio compartido).
  • Cableado propio del usuario.
  • Diseñadas inicialmente para datos.
  • Alta velocidad (Mbps a Gbps) y baja latencia.

Topologías Físicas vs Lógicas

Topología Física: cómo están conectadas las computadoras al medio físico (cómo va el cable).
Topología Lógica: cómo el computador accede al medio (quién transmite y cuándo).

Topologías físicas

Topologías lógicas

  • Broadcast: cada host envía sus datos a todos. Acceso "first come, first serve". Ethernet trabaja así.
  • Token Passing: el acceso al medio se controla con un "token" electrónico. El que tiene el token transmite. Evita colisiones.

Redes WAN (Wide Area Network)

Utilizan medios telefónicos (diseñados inicialmente para voz). Son servicios contratados a operadoras (Claro, Movistar, ETB). Costos elevados, se diseñan optimizadamente.

Servicios de comunicación WAN

  • Líneas dedicadas: enlace dedicado permanente con caudal reservado, se use o no.
  • Conmutación de circuitos: conexión se establece cuando se necesita; caudal reservado mientras dura. Ej: PSTN, ISDN.
  • Conmutación de paquetes: ancho de banda compartido entre varios usuarios. Tráfico multiplexado. Más eficiente. Ej: Frame Relay, ATM, MPLS.

Redes punto a punto: modos de transmisión

  • Simplex: transmisión en un solo sentido. Ej: TV, radio.
  • Half-duplex (semi-dúplex): ambos sentidos, pero no a la vez. Ej: radios walkie-talkie.
  • Full-duplex (dúplex): transmisión simultánea en ambos sentidos. Ej: teléfono, Ethernet moderna.
Para el examen recuerda: LAN = transporte de datos diseñado para eso. WAN = adaptación de redes telefónicas (voz) para datos. La topología Ethernet es BROADCAST lógica aunque físicamente sea estrella.

Tema 2 — Modelo OSI y Pila TCP/IP Modelos de capas

Todas las arquitecturas de red modernas se describen con modelos de capas. Los dos más importantes son OSI (7 capas) de ISO y TCP/IP (4 capas) de DARPA.

Origen histórico

  • ISO crea OSI en 1984 — International Organization for Standardization → Open Systems Interconnection. Un modelo de referencia teórico.
  • DARPA crea ARPANET como red de respaldo ante un ataque nuclear → de ahí nace TCP/IP, el modelo práctico que usa Internet hoy.

¿Por qué un modelo en capas?

Dividir el problema en capas:
  • Reduce la complejidad
  • Estandariza las interfaces
  • Facilita la ingeniería modular
  • Garantiza tecnología interoperable
  • Acelera la evolución
  • Simplifica la enseñanza y el aprendizaje

Principios del modelo de capas

  • La capa n ofrece sus servicios a la capa n+1. La capa n+1 solo usa los servicios de la capa n.
  • La comunicación entre capas se realiza mediante una interfaz.
  • Cada capa se comunica con la capa equivalente en el otro sistema usando un protocolo característico de esa capa.
  • El protocolo forma parte de la arquitectura, la interfaz no.
Como una traducción entre artistas: artista ruso ↔ traductor → inglés → traductor ↔ artista valenciano. Cada nivel habla su "idioma" (protocolo) con su par del otro lado.

Capas de Host vs Capas de Medio

  • Capas de Hosts (4-7): Aplicación, Presentación, Sesión, Transporte. Proporcionan entrega precisa de datos entre computadores.
  • Capas de Medios (1-3): Red, Enlace de Datos, Física. Controlan la entrega física de mensajes a través de la red.

Detalle por capa OSI

Capa 7 — Aplicación

Provee los servicios de red a las aplicaciones del usuario. Permite que apps como Word "vean" los recursos de red.

Protocolos: HTTP (80), FTP (20/21), SMTP (25), DNS (53), TFTP (69), POP3, Telnet (23), SNMP (161)

Capa 6 — Presentación

Representación de datos y formateo de códigos. Garantiza que los datos sean entendidos por la aplicación del otro lado.

Funciones: Compresión, cifrado/encripción, formato de datos (ASCII, EBCDIC).

Capa 5 — Sesión

Establece, mantiene y termina sesiones entre hosts. Usa "dialog control" y "dialog separation".

Protocolos: NFS, SQL, RPC, X-Window, ASP, SCP

Capa 4 — Transporte

Provee confiabilidad, control de flujo y corrección de errores. Segmenta datos, agrega headers con secuencia y confirmación. Maneja puertos origen/destino.

Protocolos: TCP (orientado a conexión, confiable) y UDP (no orientado a conexión, poco confiable).

Capa 3 — Red

Responsable del direccionamiento lógico del paquete y la determinación de ruta.

  • Addressing → protocolos enrutados (IP, IPX, AppleTalk, DECnet)
  • Path Selection → protocolos de enrutamiento (RIP, IGRP, EIGRP, OSPF, BGP)

Routers operan en esta capa.

Capa 2 — Enlace de Datos

Provee acceso al medio. Maneja notificación de errores, topología de red, control de flujo y direccionamiento físico de la trama (frame).

Media Access Control vía:

  • Determinística — token passing (Token Ring, FDDI)
  • No-determinística — broadcast topology, dominios de colisión (Ethernet/CSMA-CD)

Capa 1 — Física

Proporciona los medios eléctricos, mecánicos, de procedimiento y funcionales para activar y mantener el enlace físico entre sistemas.

Medios: Cable CAT 5, cable coaxial, fibra óptica, atmósfera (inalámbrico).

Cada capa añade su propio "encabezado" (header) a los datos. En el destino, cada capa quita su encabezado correspondiente.

Envío de un email:

2. Presentación los codifica en ASCII.

Tema 3 — Capa de Enlace de Datos Capa 2 OSI

La Capa 2 es donde los bits "crudos" se convierten en tramas con sentido. Es la frontera entre el mundo físico (cables, señales) y el mundo lógico (paquetes, IPs).

¿Por qué existe la Capa 2?

La Capa 1 (Física) tiene limitaciones serias:

  • No puede organizar flujos de bits.
  • No puede nombrar o identificar computadoras.
  • No puede comunicarse con capas superiores.
  • No puede decidir qué computadora transmite.

La Capa 2 resuelve todo esto.

Funciones principales

  • Framing: organiza los bits en grupos llamados tramas (frames).
  • Addressing: identifica computadoras mediante direcciones (MAC).
  • Logical Link Control (LLC): se comunica con las capas superiores.
  • Media Access Control (MAC): decide quién transmite y cuándo.

Las dos sub-capas: LLC y MAC

La Capa 2 IEEE se divide en dos:
  • LLC (802.2): Logical Link Control — interfaz con Capa 3, control de flujo y errores.
  • MAC (802.3, 802.5...): Media Access Control — direccionamiento físico (MAC address) y acceso al medio.

Direcciones MAC

  • Identificador único de 48 bits (6 bytes) grabado en la tarjeta de red.
  • Formato hexadecimal: 00:1A:2B:3C:4D:5E o 001A.2B3C.4D5E.
  • Primeros 24 bits = OUI (Organizationally Unique Identifier, identifica al fabricante).
  • Últimos 24 bits = número de serie del dispositivo.
  • MAC de broadcast: FF:FF:FF:FF:FF:FF (todos los bits a 1).

Protocolos MAC: Determinístico vs No-determinístico

Determinístico — "Tomemos turnos"

Cada estación espera su turno. Predecible, sin colisiones.

  • Token Ring (IEEE 802.5): un token circula entre estaciones. Solo el que tiene el token puede transmitir.
  • FDDI: dos anillos de fibra. También usa token passing.

No-determinístico — "El primero que llegue, el primero servido"

  • Ethernet (IEEE 802.3): usa CSMA/CD (Carrier Sense Multiple Access with Collision Detection).

CSMA/CD paso a paso

Carrier Sense: antes de transmitir, escucho el medio. ¿Hay alguien transmitiendo?
Multiple Access: todos pueden acceder al medio.
Collision Detection: mientras transmito, detecto si hubo colisión. Si sí, espero un tiempo aleatorio (backoff) y reintento.
  1. Estación quiere transmitir → escucha el medio.
  2. Si está libre → transmite.
  3. Si dos transmiten a la vez → colisión.
  4. Detectada la colisión → envían "jam signal" para que todos se enteren.
  5. Esperan un tiempo aleatorio (backoff exponencial binario).
  6. Reintentan.

Dominios de colisión vs dominios de broadcast

DispositivoDominios de colisiónDominios de broadcast
Hub1 (todos comparten)1
Switch1 por puerto1
Router1 por puerto1 por puerto

Formato general de una trama

+----------+----------+----------+--------+----------+--------+
| Preamble | Dst MAC  | Src MAC  | Type   | Datos    | FCS    |
+----------+----------+----------+--------+----------+--------+
  • Preamble: sincronización.
  • Dst MAC / Src MAC: direcciones MAC destino y origen.
  • Type: qué protocolo de capa superior se usa (IP, ARP, etc.).
  • Datos: payload (el paquete IP).
  • FCS (Frame Check Sequence): CRC para detectar errores.

Dispositivos de Capa 2

  • NIC (tarjeta de red): donde está la MAC.
  • Bridge: conecta dos segmentos. Decisión basada en MAC.
  • Switch: "bridge multipuerto". Aprende MACs, crea tabla CAM, envía tramas solo al puerto destino.
El examen suele preguntar: "¿Qué capa direcciona con MAC?" Respuesta: Capa 2. "¿Qué capa segmenta el medio en dominios de colisión separados?" Respuesta: Capa 2 (switches). "¿Qué capa hace lo mismo con broadcast?" Respuesta: Capa 3 (routers).

Tema 4 — Tecnología Ethernet IEEE 802.3

Ethernet es la tecnología LAN más usada del mundo. Domina porque es barata, simple y escalable. Aquí está todo lo que necesitas saber.

Historia y estándar

  • Desarrollada por Xerox PARC en los 70s (Robert Metcalfe, 1973).
  • Estandarizada como IEEE 802.3 en 1983.
  • Topología lógica: broadcast. Topología física: bus (antiguo) o estrella (moderno).
  • Acceso al medio: CSMA/CD.

100 = velocidad en Mbps

TX = tipo de medio (T = twisted pair, F = fiber)

Trama Ethernet (IEEE 802.3)

+----------+-----+----------+----------+--------+----------+--------+
| Preamble | SFD | Dst MAC  | Src MAC  | Len/Tp | Datos    | FCS    |
|  7 bytes | 1 B | 6 bytes  | 6 bytes  | 2 B    |46-1500 B | 4 B    |
+----------+-----+----------+----------+--------+----------+--------+
  • Preamble (7 bytes): patrón 10101010 para sincronización.
  • SFD (1 byte): Start Frame Delimiter, 10101011. Marca dónde empieza la trama.
  • Destination MAC (6 bytes): MAC destino.
  • Source MAC (6 bytes): MAC origen.
  • Length/Type (2 bytes): longitud o tipo de protocolo superior (0x0800 = IP).
  • Datos (46-1500 bytes): payload. Si es menor a 46, se rellena con padding.
  • FCS (4 bytes): Frame Check Sequence — CRC-32 para detectar errores.

MTU (Maximum Transmission Unit)

El tamaño máximo de datos en una trama Ethernet estándar es 1500 bytes. Esta es la MTU típica. Jumbo frames pueden llegar a 9000 bytes en redes que lo soporten.

Dispositivos Ethernet

DispositivoCapaFunción
Repetidor1Regenera la señal eléctrica
Hub1Repetidor multipuerto. Todos los puertos comparten el mismo dominio de colisión
Bridge2Conecta dos segmentos. Aprende MACs
Switch2Bridge multipuerto. Un dominio de colisión por puerto
Router3Conecta redes diferentes

Cómo aprende un switch

  1. Switch inicia con tabla MAC vacía.
  2. Llega trama → mira MAC origen → la asocia al puerto por donde entró → la guarda en su tabla CAM.
  3. Mira MAC destino → busca en su tabla.
    • Si encuentra el puerto → envía solo por ese puerto (forwarding).
    • Si NO la encuentra → envía por todos los puertos excepto el de origen (flooding).
  4. Si la MAC destino es broadcast (FFFF.FFFF.FFFF) → siempre flooding.

Tipos de transmisión en switch

  • Store-and-forward: recibe la trama completa, verifica FCS, luego reenvía. Más seguro, más lento.
  • Cut-through: apenas lee la MAC destino, reenvía. Más rápido, no verifica errores.
  • Fragment-free: espera los primeros 64 bytes (donde suelen ocurrir colisiones), luego reenvía.

Half-duplex vs Full-duplex

  • Half-duplex: transmite o recibe, no ambos. Usa CSMA/CD. Típico con hubs.
  • Full-duplex: transmite y recibe simultáneamente. NO usa CSMA/CD. Sin colisiones. Típico con switches.

Cables Ethernet

CableUso
Straight-through (directo)Dispositivos distintos: PC↔Switch, Switch↔Router
Crossover (cruzado)Dispositivos iguales: PC↔PC, Switch↔Switch, Router↔Router
Rollover (consola)PC↔puerto consola de router/switch (configuración)
Los switches modernos tienen Auto-MDIX: detectan el tipo de cable automáticamente. En la práctica casi nunca importa, pero en el examen sí lo preguntan.
Recuerda: la velocidad de Ethernet no aumenta tu velocidad de Internet si tu enlace WAN es más lento. Ethernet a 1 Gbps con Internet de 50 Mbps → tu Internet sigue siendo 50 Mbps.

Tema 5 — Direccionamiento IP IPv4 + Subnetting

Este es el tema más numérico del curso. Si dominas direccionamiento IP, dominas redes.

¿Qué es una dirección IP?

Identificador lógico (de software) de Capa 3 que identifica un dispositivo en una red. IPv4 usa 32 bits agrupados en 4 octetos de 8 bits, expresados en decimal: 192.168.1.1.

Estructura de una IP

Una IP se divide en dos partes:

  • Network portion (red): identifica la red.
  • Host portion (host): identifica al dispositivo dentro de la red.

La máscara de subred (subnet mask) dice cuántos bits son de red.

Conversión binaria-decimal (lo que necesitas saber)

Posición12345678
Valor1286432168421

Un octeto de 8 bits puede valer de 0 (00000000) a 255 (11111111). Cada bit en 1 suma su valor.

11000000 = 128 + 64 = 192
10101000 = 128 + 32 + 8 = 168
11111111 = 128+64+32+16+8+4+2+1 = 255

Fórmulas clave

Hosts útiles = 2(bits de host) − 2
Se restan 2 porque la primera dirección es la red y la última es el broadcast.

Subredes posibles = 2(bits prestados)

Subnetting paso a paso

Ejemplo: tengo 192.168.1.0/24 y necesito crear 4 subredes.

Paso 1: Calcular bits a prestar

Necesito 4 subredes. 2n ≥ 4 → n = 2 bits. Le presto 2 bits a la porción de host.

Paso 2: Nueva máscara

Original: /24. Más 2 bits = /26. Máscara: 255.255.255.192.

Paso 3: Calcular tamaño de cada subred

Bits de host = 32 − 26 = 6 bits. Hosts totales = 26 = 64. Hosts útiles = 64 − 2 = 62.

Wildcard mask

La inversa de la máscara. Se usa en ACLs y OSPF.

Máscara 255.255.255.0 → Wildcard 0.0.0.255
Máscara 255.255.255.252 → Wildcard 0.0.0.3
Truco: 255 - octeto de máscara = octeto de wildcard.

VLSM (Variable Length Subnet Masking)

Permite usar máscaras de distinto tamaño en distintas subredes. Más eficiente que subnetting tradicional.

De 192.168.1.0/24:
• LAN A (100 hosts) → /25 (126 útiles): 192.168.1.0 – 192.168.1.127
• LAN B (50 hosts) → /26 (62 útiles): 192.168.1.128 – 192.168.1.191
• Enlace serial → /30 (2 útiles): 192.168.1.192 – 192.168.1.195

IPv6 (vista panorámica)

  • 128 bits (vs 32 de IPv4) → 340 sextillones de direcciones.
  • Notación hexadecimal: 2001:0db8:acad:1::1/64.
  • Sin NAT, sin broadcast (usa multicast).
  • Coexistencia con IPv4 → "dual stack".

Funciones de un router

  • Conectar redes separadas.
  • Mejor ruta basada en Capa 3.
  • Switching de paquetes: recibe por una interfaz, envía por la apropiada.
  • Segmentar dominios de broadcast.
  • Filtrar tráfico (con ACLs).

Comunicación entre redes

Cuando una PC quiere comunicarse con otra:

  1. Compara su IP con la del destino aplicando su máscara.
  2. Si están en la misma red → envía directo (ARP para encontrar MAC).
  3. Si están en redes distintas → envía al gateway (router) y este se encarga.

ARP (Address Resolution Protocol)

Traduce IP → MAC. Esencial para comunicación en LAN.

  1. PC1 quiere enviar a 192.168.1.10. ¿Cuál es su MAC?
  2. PC1 manda ARP request en broadcast: "¿Quién tiene 192.168.1.10?"
  3. La PC que tiene esa IP responde con su MAC.
  4. PC1 guarda en su tabla ARP la asociación IP-MAC.

ICMP (Internet Control Message Protocol)

Protocolo de mensajes de control y error en Capa 3. Lo usa ping y traceroute.

  • Echo Request / Echo Reply: ping.
  • Destination Unreachable: destino inalcanzable.
  • Time Exceeded: TTL llegó a 0. Usado por traceroute.
  • Redirect: mejor ruta disponible.

Encabezado IPv4 (resumen)

  • Version: 4 o 6.
  • TTL (Time To Live): contador que se decrementa en cada router. Si llega a 0 → paquete descartado. Evita loops.
  • Protocol: qué protocolo de capa superior (TCP=6, UDP=17, ICMP=1).
  • Source IP / Destination IP.
  • Header Checksum: verificación.

NAT (Network Address Translation)

Permite que múltiples dispositivos con IPs privadas salgan a Internet con una sola IP pública.

  • NAT estático: 1 IP privada ↔ 1 IP pública.
  • NAT dinámico: pool de IPs públicas asignadas dinámicamente.
  • PAT (NAT overload): muchas IPs privadas comparten 1 IP pública usando puertos. Lo más común.
Recuerda: routers separan dominios de broadcast. Switches solo dominios de colisión. Si una red tiene demasiado tráfico de broadcast, hay que dividirla con routers o VLANs.

Tema 7 — Algoritmos de Enrutamiento Cómo eligen las rutas

Los protocolos de enrutamiento usan algoritmos para calcular el mejor camino. Conocer estos algoritmos te ayuda a entender por qué OSPF, RIP, EIGRP y BGP funcionan distinto.

Clasificación de protocolos de enrutamiento

Por su alcance

  • IGP (Interior Gateway Protocol): dentro de un sistema autónomo. Ej: RIP, OSPF, EIGRP, IS-IS.
  • EGP (Exterior Gateway Protocol): entre sistemas autónomos. Ej: BGP.

Por su algoritmo

  • Distance Vector: "router rumor" — cada router le cuenta a sus vecinos qué redes conoce y a qué distancia.
  • Link State: "mapa completo" — cada router conoce la topología completa y calcula rutas con Dijkstra.
  • Hybrid: mezcla. Ej: EIGRP.
  • Path Vector: usa caminos completos. Ej: BGP.

Distance Vector — "Routing by rumor"

Cada router envía periódicamente su tabla de routing completa a sus vecinos directos. Los vecinos actualizan su propia tabla. Eventualmente todos convergen.

Características

  • Actualizaciones periódicas (cada 30s en RIP).
  • Solo intercambian con vecinos directos.
  • Lento para converger.
  • Métricas simples (hops).
  • Vulnerable a loops.

Algoritmo Bellman-Ford

Base matemática del distance vector. Cada router actualiza su distancia a un destino usando la fórmula: D(X→Z) = mín{C(X→Y) + D(Y→Z)} para todos los vecinos Y.

Problema: routing loops

Si un router cree que para llegar a la red A debe pasar por router B, y B cree que debe pasar por A → loop infinito.

Soluciones a los loops

  • Maximum hop count: RIP usa máximo 15 hops. Lo que esté a 16 es "infinito" (inalcanzable).
  • Split Horizon: no anuncies una ruta de regreso por la misma interfaz por donde la aprendiste.
  • Route Poisoning: cuando una ruta cae, anúnciala con métrica "infinita" para que todos sepan.
  • Hold-down timers: cuando una ruta cae, espera antes de aceptar nuevas actualizaciones sobre ella.
  • Triggered updates: cuando algo cambia, manda actualización inmediata sin esperar al timer.

Ejemplos de protocolos distance vector

  • RIP v1/v2: métrica = hops. Máx 15. Cada 30s. Simple, lento, poco escalable.
  • IGRP: propietario de Cisco. Métrica compuesta (ancho de banda, retardo, carga, confiabilidad). Obsoleto.

Link State — "Mapa completo de la red"

Cada router conoce la topología completa de la red. Envía información solo cuando hay cambios (no periódicamente). Usa el algoritmo de Dijkstra (SPF — Shortest Path First) para calcular rutas.

Pasos del algoritmo Link State

  1. Cada router descubre a sus vecinos directos usando paquetes Hello.
  2. Cada router envía LSAs (Link State Advertisements) describiendo sus enlaces.
  3. Los LSAs se inundan (flooding) por toda el área.
  4. Cada router construye una LSDB (Link State Database) completa.
  5. Cada router ejecuta Dijkstra sobre su LSDB para construir el árbol SPF.
  6. Del árbol SPF derivan las rutas para la tabla de routing.

Características

  • Convergencia rápida.
  • Solo envía actualizaciones cuando hay cambios (incrementales).
  • No vulnerable a loops como distance vector.
  • Más uso de CPU/memoria.
  • Métrica = costo (basado en ancho de banda).

Algoritmo de Dijkstra (SPF)

  1. Marco mi router como "actual", costo 0.
  2. Examino los vecinos del actual, calculo costo total a cada uno.
  3. Elijo el de menor costo, lo marco como visitado.
  4. Repito desde ese nodo hasta visitar todos.
  5. Resultado: árbol de caminos más cortos desde mi router a todos los destinos.

Ejemplos de protocolos link state

  • OSPF (Open Shortest Path First): el rey de los IGP. Estándar abierto. Áreas, router-IDs, áreas backbone (area 0).
  • IS-IS: similar a OSPF. Usado en redes de proveedores grandes.

Híbrido: EIGRP

Combina lo mejor de distance vector y link state. Propietario de Cisco (ahora abierto). Usa el algoritmo DUAL (Diffusing Update Algorithm) para garantizar rutas libres de loops y convergencia rápida.

  • Métrica compuesta: ancho de banda + retardo (por defecto).
  • Mantiene successor (mejor ruta) y feasible successor (backup).
  • Convergencia muy rápida.

Path Vector: BGP

Usado entre sistemas autónomos en Internet. No usa métricas simples sino políticas (atributos). El "AS-Path" registra los AS por los que pasó la ruta.

Comparación rápida

Convergencia

Tiempo que tardan todos los routers en tener una visión consistente de la red después de un cambio. Cuanto más rápida la convergencia, mejor.

Métricas

  • Hop count: número de routers entre origen y destino. Simple pero engañosa.
  • Bandwidth: ancho de banda del enlace.
  • Delay: retardo del enlace.
  • Cost: número calculado, normalmente inverso al ancho de banda.
  • Load: utilización actual del enlace.
  • Reliability: errores en el enlace.
Para el parcial recuerda: OSPF es link state, usa Dijkstra, área 0 es la backbone, métrica = costo, AD = 110. Es lo que vas a configurar en la práctica.

Tema 8 — Enrutamiento Estático Rutas manuales

Antes de usar OSPF, hay que entender el enrutamiento manual: las rutas estáticas. Son la base y siguen siendo útiles en muchos casos reales.

¿Qué es una ruta estática?

Una ruta que el administrador configura manualmente en el router. No se aprende, no se intercambia con otros routers, no cambia salvo que tú la cambies.

Ventajas y desventajas

VentajasDesventajas
Sin overhead de CPU/ancho de bandaConfiguración manual en cada router
Predecible, fácil de auditarNo se adapta a cambios
Más segura (no anuncia rutas)No escala en redes grandes
Útil para enlaces stub o punto-a-puntoMantenimiento alto

Cuándo usar rutas estáticas

  • Redes muy pequeñas (3-4 routers).
  • Conexión a un ISP (ruta por defecto).
  • Redes "stub" (sin redundancia).
  • Routers que no soportan protocolos dinámicos.
  • Como backup de rutas dinámicas con AD más alta ("floating static route").

Sintaxis del comando

R1(config)# ip route <red_destino> <máscara> { <next-hop> | <interfaz_salida> } [distancia]

Tipos de rutas estáticas

1. Ruta estática estándar (next-hop)

R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2

"Para llegar a la red 192.168.2.0/24, manda el paquete a la IP 10.0.0.2".

2. Ruta estática por interfaz de salida

R1(config)# ip route 192.168.2.0 255.255.255.0 Serial0/0/0

"Para llegar a esa red, manda por mi interfaz Serial0/0/0".

Usar interfaz de salida es típico en enlaces seriales punto-a-punto. Con interfaces broadcast (Ethernet) es preferible usar next-hop para que ARP sepa a quién resolver.

3. Ruta completamente especificada

R1(config)# ip route 192.168.2.0 255.255.255.0 Serial0/0/0 10.0.0.2

Especifica AMBAS: interfaz y next-hop. Más explícito.

4. Ruta por defecto (default route)

R1(config)# ip route 0.0.0.0 0.0.0.0 Serial0/0/1

"Para cualquier destino desconocido, manda por Serial0/0/1". Útil para enlace a ISP.

Esto es exactamente lo que configuras en el Parcial OSPF: la ruta por defecto desde R1 hacia el ISP, y luego la rediriges por OSPF con default-information originate.

5. Ruta estática flotante (floating static)

R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2 200

Le ponemos AD alta (mayor que la del protocolo dinámico) para que NO se use a menos que el camino dinámico caiga. Es un backup.

Verificación

R1# show ip route static    ! Solo rutas estáticas
R1# show ip route             ! Todas las rutas
R1# show running-config | section ip route

Eliminar una ruta estática

R1(config)# no ip route 192.168.2.0 255.255.255.0 10.0.0.2

Ejemplo completo: red de 3 routers

Topología:
[LAN1 192.168.1.0/24] — R1 — [10.0.0.0/30] — R2 — [20.0.0.0/30] — R3 — [LAN3 192.168.3.0/24]

Cada router necesita rutas a las redes que NO tiene directamente conectadas.

En R1

R1(config)# ip route 20.0.0.0 255.255.255.252 10.0.0.2
R1(config)# ip route 192.168.3.0 255.255.255.0 10.0.0.2

En R2

R2(config)# ip route 192.168.1.0 255.255.255.0 10.0.0.1
R2(config)# ip route 192.168.3.0 255.255.255.0 20.0.0.2

En R3

R3(config)# ip route 10.0.0.0 255.255.255.252 20.0.0.1
R3(config)# ip route 192.168.1.0 255.255.255.0 20.0.0.1

Sumarización (ruta resumida)

En vez de configurar varias rutas estáticas, usa una sola que cubra todas las subredes.

En vez de:
ip route 192.168.0.0 255.255.255.0 ...
ip route 192.168.1.0 255.255.255.0 ...
ip route 192.168.2.0 255.255.255.0 ...
ip route 192.168.3.0 255.255.255.0 ...

Una sola ruta resumida:
ip route 192.168.0.0 255.255.252.0 ...

Errores comunes

  • ❌ Olvidar configurar la ruta en AMBOS sentidos. La ida funciona, la vuelta no.
  • ❌ Apuntar a un next-hop que el router no puede alcanzar.
  • ❌ Máscara incorrecta en la ruta.
  • ❌ Usar interfaz de salida en Ethernet sin next-hop → ARP problems.
  • ❌ Crear loops por rutas mal configuradas.
Para el parcial: la única ruta estática que vas a configurar es la ruta por defecto en R1 hacia el ISP (ip route 0.0.0.0 0.0.0.0 Serial0/0/1). Las demás rutas las aprenderán los routers automáticamente por OSPF.

Routers vs Switches Cisco

Tienen muchas similitudes:

  • Sistema operativo modal similar (IOS).
  • Estructura de comandos similar.
  • Muchos comandos compartidos.
  • Pasos de configuración inicial idénticos.

Diferencias clave: las interfaces. Switches LAN (FastEthernet, GigabitEthernet). Routers además tienen Serial, DSL, módulos HWIC.

Topología Dual Stack

"Dual Stack" significa que el router maneja IPv4 e IPv6 simultáneamente en las mismas interfaces. Se configuran ambas direcciones y el router enruta según el tipo de paquete.

Interfaces Loopback

Interfaz lógica, interna al router. No conectada a nada físico. Siempre está en estado "up" mientras el router funcione.

¿Para qué sirven?

  • Probar y gestionar el router.
  • Simular redes detrás del router (en laboratorios).
  • Identificar al router en protocolos como OSPF (router-ID).
  • Simular un enlace a Internet en prácticas.

Configuración

R1(config)# interface loopback 0
R1(config-if)# ip address 10.0.0.1 255.255.255.0

No necesitas no shutdown — las loopbacks vienen activas por defecto.
